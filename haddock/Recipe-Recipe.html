type Recipe = Tree Action

Recipe type is simply a Data.Tree of Actions.

data Action

Actions represent something that is done during the recipe and are taken to be "applied" on their child nodes in the recipe thus providing sequencing. Not used directly, used via the helper functions.

Combine the two recipes using the method given as a String e.g. "mix".

addCondition :: Condition -> Recipe -> Recipe

Add the given condition to the root action of the given recipe. If the action is already wrapped with a condtion, (AND) is applied to the two conditions.

Wraps the root node of a recipe in a Transaction.

Adds a Measure action with the given Measurement above the given recipe.

foldCond :: (Ord a, Monoid a) => (Condition -> a) -> Condition -> a

Folds a function over a condition. AND will mappend the values from folding over the two conditions whereas OR takes the max.
